# House Price Prediction

---

Проект по регрессионному машинному обучению, посвящённый **прогнозированию цен продажи домов**
на основе структурированных табличных данных.

**Языки:** [English](README.md) | [Русский](README_RU.md)

---

## Кратко о проекте

Проект, охватывающий полный ML-цикл, по прогнозированию цен на дома — от исследовательского анализа и сравнения моделей
до воспроизводимого пайплайна обучения и FastAPI-сервиса.

- Структурированный ноутбук с EDA, статистическим анализом, анализом связей признаков и сравнением моделей
- Финальная инженерная реализация на основе регуляризованной линейной модели (Lasso), выбранной по итогам экспериментов
- Единый пайплайн предобработки для обучения и инференса, а также структурное логирование
- FastAPI-сервис для онлайн-предсказаний
- Автотесты на pytest и CI через GitHub Actions

---

## Быстрый старт

В репозитории есть **две части**:

1. **Notebook / research layer** — исследовательский анализ, эксперименты и выбор финальной модели  
2. **Code implementation layer** — воспроизводимый train-скрипт, predict-скрипт, API и тесты

Исследовательскую часть можно отдельно открыть и изучить в ноутбуке:

```text
notebooks/house-price-predictor.ipynb
```

Команды ниже относятся к кодовой реализации проекта и позволяют:

- обучить финальную модель;
- получить предсказания для тестового набора Kaggle;
- запустить FastAPI-сервис локально для онлайн-предсказаний.

**Требования:**

- Python 3.9+ (проверено на Python 3.12)
- Kaggle API token для загрузки датасета

### 1) Клонировать репозиторий

```bash
git clone https://github.com/lid11a/house-price-predictor.git
cd house-price-predictor
```

### 2) Настроить Kaggle API token

- **Windows:** `%USERPROFILE%\.kaggle\kaggle.json`
- **Linux / macOS:** `~/.kaggle/kaggle.json`

Для Linux/macOS нужно выставить права:

```bash
chmod 600 ~/.kaggle/kaggle.json
```

### 3) Создать виртуальное окружение и установить зависимости

**Windows (PowerShell)**

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

**macOS / Linux**

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 4) Обучить модель

```bash
python src/train.py
```

### 5) Получить предсказания для тестового датасета `test.csv`

```bash
python src/test.py
```

### 6) Поднять API локально

```bash
uvicorn api.main:app --app-dir src --host 0.0.0.0 --port 8000
```

Swagger UI:

```text
http://127.0.0.1:8000/docs
```

### 7) Запустить тесты и CI (GitHub Actions)

Локально тесты можно запустить командой:

```
pytest
```

GitHub Actions workflow находится в:

```
.github/workflows/ci.yml
```

Пайплайн автоматически запускается при:

- push;
- pull_request.

В рамках CI выполняются следующие шаги:

- checkout репозитория;
- настройка Python 3.11;
- установка зависимостей;
- запуск pytest.

## Обзор проекта

Проект показывает полный workflow для задачи табличной регрессии:
от EDA и статистического анализа до сравнительного моделирования и
воспроизводимой реализации для обучения и инференса.

Проект специально разделён на две взаимодополняющие части.

### 1) Исследовательская / notebook-часть

Реализована в:

```text
notebooks/house-price-predictor.ipynb
```

Здесь находится аналитическая работа, на которой основаны выводы проекта.
Она включает:

- анализ распределения таргета и обоснование log-трансформации;
- анализ числовых и категориальных признаков;
- корреляционный анализ Pearson / Spearman;
- анализ связи категориальных признаков с таргетом;
- сравнение нескольких семейств регрессионных моделей;
- интерпретацию поведения моделей и выбор финального решения.

В ноутбуке исследуются и сравниваются несколько семейств моделей, включая:

- линейные модели;
- регуляризованные линейные модели;
- полиномиальную регрессию;
- distance-based модели;
- модели на деревьях и ансамблях;
- gradient boosting модели.

По итогам экспериментов делается практический вывод:
регуляризованный линейный подход даёт лучший баланс между качеством,
простотой и устойчивостью на этом датасете.  
Именно этот выбор и перенесён в кодовую часть проекта.

То есть ноутбук показывает **полный исследовательский путь и сравнение моделей**,
а кодовая база реализует **финальный выбранный workflow**.

### 2) Кодовая / инженерная часть

Реализована в `src/` и `tests/`.

Эта часть переводит выводы из ноутбука в компактную воспроизводимую ML-реализацию.
Она включает:

- воспроизводимую загрузку датасета из Kaggle;
- скрипт обучения финальной модели;
- сохранение артефакта модели и метрик;
- скрипт предсказания для тестового набора;
- FastAPI-сервис для онлайн-предсказаний;
- эндпоинты с описанием входных признаков;
- автотесты и CI.

Во время обучения сохраняются:

- `artifacts/model.joblib`
- `artifacts/metrics.json`

В артефакте лежат обученная модель и метаданные, включая target column и версию scikit-learn.

#### API-сервис

FastAPI-приложение реализовано в:

```text
src/api/main.py
```

Доступные эндпоинты:

- `GET /health` — проверка, что сервис жив
- `GET /schema` — machine-readable схема входа
- `GET /cheatsheet` — markdown-памятка по признакам и категориям
- `POST /predict` — онлайн-предсказание для одной или нескольких записей

API использует тот же сохранённый артефакт, что и batch prediction script.

#### Логирование

Для script-based workflow настроено структурированное логирование.

Файлы логов:

- `logs/train.log`
- `logs/predict_batch.log`

#### Тесты и quality control

В проекте есть автоматические тесты для ключевых частей реализации.

Сейчас тесты покрывают:

- корректность работы API;
- вспомогательную логику моделирования;
- механизм обеспечения совместимости со scikit-learn.

Тесты запускаются через `pytest` как локально, так и в CI.

│   │   ├── __init__.py             # пакет API
│   │   └── main.py                 # FastAPI-приложение: /health, /schema, /cheatsheet, /predict
│   ├── house_price_predictor/
│   │   ├── __init__.py             # пакет с основной логикой проекта
│   │   ├── config.py               # конфигурация и пути (data, artifacts, logs и т.д.)
│   │   ├── data.py                 # утилиты для загрузки и чтения данных (CSV)

│   │   ├── modeling.py             # построение pipeline и вспомогательные функции моделирования

│   │   └── sklearn_compat.py       # патч совместимости с версиями scikit-learn
│   ├── train.py                   # CLI-скрипт обучения модели и сохранения артефактов

├── pytest.ini                     # конфигурация pytest (пути, настройки)
├── README.md                      # описание проекта (английская версия)
├── README_RU.md                   # описание проекта (русская версия)

## Данные

Проект использует датасет из Kaggle competition:

```text
house-prices-advanced-regression-techniques
```

Сырые данные не хранятся в репозитории.
Они скачиваются локально через Kaggle CLI.

Локальные пути по умолчанию:

- `data/raw/train.csv`
- `data/raw/test.csv`
- `data/raw/data_description.txt`

Если `train.csv` и `test.csv` уже лежат локально, повторное скачивание не выполняется, кроме случая с `--force-download`.
